1. Descrição do Projeto

Projeto de integração entre dois sistemas utilizando Firebase Realtime Database como camada de interoperabilidade. O Sistema A (Python) atua como pipeline ETL que valida, transforma e carrega dados. O Sistema B (Flutter/Dart) é um aplicativo mobile que consome esses dados e oferece uma loja virtual completa com autenticação, carrinho e pedidos.

Sem APIs pagas. O projeto utiliza exclusivamente o plano gratuito do Firebase (Spark Plan: autenticação ilimitada, 1 GB de banco de dados, 10 GB/mês de transferência).

3. Sistema A — Pipeline ETL (Python)

Arquivo: seed_firebase.py

Pipeline ETL em 3 etapas: Validação → Transformação → Carga.

# Agente de Validação (regras locais, sem API externa)
def validar_produto(produto):
    erros = []
    if len(produto["name"].strip()) < 3:
        erros.append("Nome inválido")
    if len(produto["description"].strip()) < 10:
        erros.append("Descrição inválida")
    if produto["price"] <= 0:
        erros.append("Preço inválido")
    return len(erros) == 0, erros

# Agente de Transformação
def transformar_produto(produto):
    return {
        "name": produto["name"].strip(),
        "price": round(float(produto["price"]), 2),
        "_migrado_em": datetime.now().isoformat(),
        "_sistema_origem": "Sistema A (Python)",
    }

# Agente de Carga — envia para Firebase (Sistema B)
def carregar_produto(produto):
    resp = requests.post(f"{FIREBASE_DB_URL}/products.json",
                         data=json.dumps(produto))
    return resp.status_code == 200

    'https://identitytoolkit.googleapis.com/v1/accounts:$urlFragment'

5. Como Reproduzir o Ambiente

Pré-requisitos

  • Python 3.10+ (python --version)
  • Flutter SDK (flutter --version)
  • Navegador Chrome ou celular Android com Depuração USB ativada

Passo a Passo

# 1. Clone o repositório
git clone https://github.com/RaphaelSilvaS/integracao-sistemas.git
cd integracao-sistemas

# 2. Execute o Sistema A (Python) — popula o Firebase
python seed_firebase.py

# 3. Instale dependências do Flutter
flutter pub get

# 4. Execute o Sistema B (Flutter) — abre no Chrome
flutter run -d chrome

# Ou em celular Android conectado via USB:
flutter run
Nenhuma chave de API paga é necessária.
Firebase Auth e Realtime Database são gratuitos no plano Spark.
